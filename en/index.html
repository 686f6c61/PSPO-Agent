From brief to backlog, no friction

A complete Product Owner workflow that lives inside Claude Code and persists everything to Markdown before publishing anywhere.

Is it in the backlog? If not, it does not exist.

Discovery and analysis

Interrogates briefs, emails or PRDs until reaching 80% clarity across 8 categories, or starts from scratch with structured questions.

Sprint-ready user stories

As / I want / So that format with Given/When/Then criteria, positive and negative scenarios, Mermaid diagrams and t-shirt estimation.

Sprint planning with AI factor

Computes the real capacity of teams working with agents (configurable 30-80% factor) and suggests cuts when the sprint overflows.

Publishing to Trello, Notion and GitHub Projects

Every story ships with a summary, attached .md file, DoD checklist, dependencies and real member assignment. No duplicates. On GitHub, cards in a private Project v2 board.

Autopilot mode

Drop a brief and a team CSV into a folder and the whole workflow runs on its own up to the final decision gate.

Security through hooks

Credentials stay out of the chat, direct API curl calls get blocked, sensitive reads raise warnings and phase gates cannot be skipped.

Install with one command

All you need is Claude Code and Python 3.8+. No npm, no pip, no build step.

Installed before the daily is over.
Linux / macOS 
curl -fsSL https://raw.githubusercontent.com/686f6c61/PSPO-Agent/main/install.sh | bash
Windows (PowerShell) 
irm https://raw.githubusercontent.com/686f6c61/PSPO-Agent/main/install.ps1 | iex
Restart Claude Code and run /pspo-agent:start. The onboarding assistant sets up the remote provider, validates credentials and gets your publishing target ready.
